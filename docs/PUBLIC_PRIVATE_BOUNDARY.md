# Public / Private Boundary

## Public

- Project description.
- Public website links.
- Selected visuals.
- Workflow diagrams.
- WordPress page draft.
- Ownership and commercial-use policy.
- High-level forum/archive/review model.

## Private

- Source code.
- WordPress theme internals.
- Deployment systems.
- Credentials and secrets.
- Prompts and agent instructions.
- Private workflows.
- Customer data.
- Legal/admin files.
- Medical or benefits records.
- Family information.
- Unpublished creative works.
- Operational infrastructure.

## Rule

The repository may explain the shape of the system. It must not expose the system.
